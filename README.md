# Что выгоднее: купить сразу или оформить рассрочку и получать проценты?

Веб-приложение, которое сравнивает два способа покупки и показывает, какой выгоднее:

1. **Оплатить сразу** — заплатить всю сумму и получить разовый бонус.
2. **Рассрочка + депозит** — положить всю сумму покупки на депозит, а беспроцентную рассрочку
   гасить из него по частям. Остаток депозита убывает и к концу срока обнуляется, но за это
   время на него капают проценты. Эти проценты (за вычетом налога) — выгода, которую
   приложение сравнивает с бонусом.

Расчёт реактивный: меняешь любое поле — вердикт, таблицы и график пересчитываются мгновенно,
без кнопки.

## Возможности

- **Входные параметры:** сумма покупки, бонус за оплату сразу, срок рассрочки (пресеты
  3 / 4 / 6 / 12 / 24 мес + произвольный), номинальная ставка депозита (по умолчанию 16%),
  капитализация (вкл/выкл), налог на доход (опционально, по умолчанию выключен — в РК
  для физлиц необязателен).
- **Крупный вердикт** — «Выгоднее депозит / оплатить сразу на +X тг».
- **Сравнительная таблица** — доход депозита до и после налога против бонуса.
- **График** — убывающий остаток депозита по месяцам.
- **Помесячная таблица** — остаток на начало / проценты / платёж / остаток на конец (раскрывается по клику).

## Стек

React 19 · TypeScript · Vite · Tailwind CSS v4 · shadcn/ui (пресет `b59i6Eu9zu`, style
`radix-nova`) · recharts · vitest. Пакетный менеджер — **pnpm**. Вся логика в браузере, без бэкенда.

## Локальный запуск

```bash
pnpm install
pnpm dev        # дев-сервер на http://localhost:5173
pnpm test       # тесты ядра расчёта (vitest)
pnpm build      # продакшен-сборка в dist/
pnpm preview    # предпросмотр собранной версии
```

## Деплой

Публикуется на **GitHub Pages** автоматически: пуш в `main` запускает workflow
[`.github/workflows/deploy.yml`](.github/workflows/deploy.yml), который ставит зависимости,
прогоняет тесты, собирает проект и выкладывает `dist/` на Pages.

Один раз нужно включить в настройках репозитория: **Settings → Pages → Build and deployment →
Source: GitHub Actions**.

В [`vite.config.ts`](vite.config.ts) задан `base: "./"` (относительные пути ассетов), поэтому
сайт работает при любом имени репозитория без правок конфига.

## Тема shadcn

Тема задаётся пресетом `b59i6Eu9zu` и хранится в виде CSS-переменных в
[`src/index.css`](src/index.css). Чтобы поменять тему — отредактируй переменные там же или
переинициализируй пресет:

```bash
npx shadcn@latest init --preset <код-пресета> --force
```

Тёмная тема использует доработанный `--primary: oklch(0.76 0.13 292)` (лавандовый),
проверенный по APCA-контрасту для читаемости на тёмном фоне.

## Структура

```
src/
  lib/
    calc.ts        # ядро расчёта (simulate, validate) — чистые функции
    calc.test.ts   # тесты ядра (vitest)
    format.ts      # форматирование сумм (тг) и парсинг ввода
    form.ts        # состояние формы и его маппинг в CalcInput
  components/
    InputPanel.tsx       # форма ввода
    VerdictCard.tsx      # крупный вердикт
    ComparisonTable.tsx  # депозит vs бонус
    BalanceChart.tsx     # график остатка (recharts)
    MonthlyTable.tsx     # помесячная таблица (collapsible)
    ui/                  # компоненты shadcn/ui
  App.tsx          # сборка: useReducer (форма) + useMemo (расчёт)
```

Подробности по архитектуре, подводным камням и правилам — в [CLAUDE.md](CLAUDE.md).
